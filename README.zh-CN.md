# CaptionAI Duo

面向 YouTube 的 AI 双语字幕扩展。它读取视频字幕轨，完成语义分句、翻译与双语对齐，再按播放器宽度和真实字体尺寸分页显示。

## 功能

- 原文与译文在同一覆盖层中显示，互不重叠。
- 默认连接 `https://api.deepseek.com`，也可直接填写其他兼容 Chat Completions 的 Base URL 和模型名。
- 支持流式 SSE 与普通 JSON 响应；流式模式会按完整语义单元逐行校验并立即显示。
- 字体、字号、颜色、背景、描边、行间距和上下顺序均可调整。
- 字幕框可上下拖动；字体、播放器尺寸或全屏状态变化时会重新分页。
- 支持 15 种目标语言、字幕开关和原文/译文/双语 SRT 导出。

## 安装

1. 打开 `chrome://extensions`。
2. 开启“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择包含 `manifest.json` 的项目目录。
5. 打开带字幕的 YouTube 视频。

需要 Chrome、Edge 或其他 Chromium 浏览器 111 及以上版本。

## 设置参数

### 翻译

| 参数 | 默认值 / 范围 | 实际含义 |
| --- | --- | --- |
| 启用 | 开启 | 控制字幕覆盖层和翻译请求。关闭时会取消当前视频的请求，并恢复由扩展代为开启的 YouTube CC。 |
| 目标语言 | 中文（简体） | AI 输出语言。改变后会清除当前视频的旧译文并按新语言重新翻译。 |
| 连接配置 | `DeepSeek` | 一套配置包含名称、Base URL、模型、推理强度、API Key 和额外请求参数。可新建、重命名、删除并从下拉框一键切换；整套配置只保存在本机，切换后当前视频按新配置重新翻译。 |
| API Base URL | `https://api.deepseek.com` | 请求地址的基址。若末尾不是 `/chat/completions`，扩展会自动补上。远程地址只允许 HTTPS；`localhost`、`127.0.0.1` 可使用 HTTP。所有受支持的 API 地址均默认放行，无需单独授权。 |
| 模型 | `deepseek-v4-flash` | 原样写入 Chat Completions 请求的 `model`。必须与所填接口支持的模型名完全一致。 |
| 推理强度 | 关闭 / `High` / `Max` | 关闭时走普通生成；DeepSeek 会收到关闭思考的字段，其他兼容接口会收到常规 `temperature`。`High`、`Max` 会发送 `reasoning_effort`，DeepSeek 同时会启用思考。接口不支持这些字段时应保持关闭。启用推理通常更慢、消耗更多，并把单次请求超时从 30 秒放宽到 90 秒。 |
| API Key | 空 | 按规范化后的 Base URL 分开保存在 `chrome.storage.local`，不会同步。请求时使用 `Authorization: Bearer …`；无鉴权的本地接口可留空，DeepSeek 必须填写。 |
| 额外请求参数 | `{}` | JSON 对象，按 Base URL 与模型分别保存在本机，用于 `enable_thinking` 等供应商专有参数。可以覆盖普通生成参数，但不能覆盖 `model`、`messages`、`stream`、Token 上限和结构化输出等核心协议字段。 |
| 上文条数 | `1`，范围 `0–20` | 取当前请求之前的若干条**原始 YouTube cue**作为只读参考，用于人名、代词、术语和语气。它们不能出现在本次译文中。滚动字幕的重复内容会先去重，并受总输入预算限制。 |
| 下文条数 | `1`，范围 `0–20` | 取当前请求之后的若干条原始 cue 作为明确标记的只读未来参考。可帮助消歧，但数值越大，模型提前知道的后续内容越多；重复内容和超出总预算的远端条目会被丢弃。 |
| 预加载批数 | `1`，范围 `0–10` | 当前播放位置之外，提前启动多少个后续预加载范围；`0` 表示关闭。这里的“批”是约 32 个词法坐标组成的调度范围，不是 32 条字幕，也不保证恰好对应一次 HTTP 请求。提高它可以减少连续播放时的等待，但会增加并发、流量和费用。 |

改变 Base URL、模型、推理强度、额外请求参数或上下文条数会使当前视频重新翻译；只改变预加载批数会取消旧的推测性预加载并按新距离继续，不会主动丢弃已经验证的译文。

### 工具

- **Token 用量**：显示当前浏览器会话中接口明确报告的输入、输出、推理、提示缓存命中/未命中，以及已报告/未报告响应数；兼容 OpenAI 的标准 `usage` 和 Gemini 的 `usageMetadata` 字段。可单独清零。本地响应缓存命中不会发起 API 请求，也不会增加 Token。
- **调试日志**：默认关闭。开启后记录字幕启动握手与恢复重试、字幕轨、紧凑请求窗口、网络尝试、Chromium `net::ERR_*` 底层传输错误、结构校验失败和分页结果。紧急请求 8 秒收不到响应头就快速重试，预取请求为 12 秒；已正常取得响应头的正文仍使用原有模型超时。日志保存在会话存储中，可复制或清空。字幕文本可能出现在日志中，提交日志前应自行检查。
- **导出 SRT**：原文直接来自已读取的完整字幕轨；译文和双语版本只导出当前已经完整翻译并通过结构校验的内容，导出动作不会为了补齐整部视频而额外翻译。

## 翻译逻辑

### 1. 获取并规范化字幕时间轴

主流程会捕获 YouTube 播放器实际请求的 `timedtext` 地址，再读取 JSON3 字幕轨，因此使用的是视频真正选中的人工字幕或自动字幕。扩展会按时间排序、补齐缺失时长，并合并滚动字幕中重复的重叠文本。若始终拿不到字幕轨，才退回到读取页面当前可见字幕的兼容路径；该路径没有完整时间轴，能力会受限。

### 2. 建立“词法坐标”，但不在本地决定语义分句

每条原始 cue 会被转换为可寻址的小单元：有空格的文本按非空白词组拆分；无空格语言使用 `Intl.Segmenter({ granularity: "word" })`，不可用时才按字符退化。若 YouTube 提供逐词偏移且能与文本精确对应，就使用真实偏移；否则在 cue 时间内按字符位置估算。

这些小单元只是时间与覆盖校验坐标，**不是最终字幕页，也不是语义边界**。相邻坐标仍可由 AI 组合成完整、不可拆的语义对齐块。`Intl.Segmenter` 的 `sentence` 模式只用于后备显示分页，不负责翻译分句。

相邻 cue 间停顿达到 900ms 时只产生软提示，AI 仍可根据语法跨越；达到 4000ms 才建立不可跨越的硬边界。YouTube cue 切换和词元窗口边缘本身都不视为句子结束。

### 3. 滚动请求与上下文

播放器附近的待翻译坐标会进入有上限的滚动窗口：基础冷窗口为 48 个词法坐标，连续处理时使用约 80 个；正在显示或刚跳转到的目标会在右侧额外预留 48 个语义展开坐标，并在窗口末尾另留 16 个私有安全坐标，避免把目标紧贴安全区而被迫重复请求。遇到更长的跨边界语义单元仍可继续扩展，但总窗口最多 160 个。当前坐标文本另有 18,000 字符上限；当前内容和去重后的上下文合计受约 28,000 字符预算约束，预算不足时优先保留当前内容及距离最近的上下文。

请求中的内容分为三类：

- `PAST_CONTEXT`：由“上文条数”控制，只读；
- `CURRENT_CUES`：本次必须按顺序完整覆盖的词法坐标；
- `FUTURE_CONTEXT`：由“下文条数”控制，只读。

为避免每个词都重复冗长字段名，当前坐标发送为紧凑的 `[id, text, pauseAfterMs, boundary]` 行，其中 boundary 为空、软边界或硬边界；上下文使用 `[id, text]`。完整 cue ID 和绝对时间仍保留在本地，用于校验、播放、分页和导出。

AI 需要先把连续坐标组合成自然句子或分句，再在每个语义单元内部返回较粗粒度的双语对齐块。上下文只能帮助理解，不能被翻译、重复或并入 `CURRENT_CUES` 的结果。

### 4. 流式传输与模型输出

请求采用兼容 Chat Completions 的 JSON，固定带 `stream: true` 和 `stream_options.include_usage: true`。主翻译协议要求模型输出 JSONL：每个已经确定的完整语义单元占一个物理行，最后只输出一行 `{"type":"done"}`。程序根据已验证单元的覆盖游标自行推导未完成后缀，模型不再重复枚举 ID，避免数字续写浪费输出 Token。DeepSeek 还会收到思考开关；JSONL 主请求不会设置只能生成单个 JSON 对象的 `response_format`。自定义兼容接口还会合并当前地址与模型对应的额外请求参数，并返回 OpenAI 风格的 `choices`。

SSE 网络块和模型 token 都可能在任意字节位置断开，因此扩展分两层缓冲：先缓冲到空行分隔的完整 SSE 事件，再把内容缓冲到完整 JSONL 换行。只有某一整行可以独立解析、ID 连续且结构完整时，该语义单元才会写入不可变缓存并立即通知页面显示；半个 JSON 字符串、半句话或任意 token 块不会直接绘制。usage-only 事件单独累计。全部当前 ID 已覆盖或 `done` 已到达后，扩展只再等待最多 750ms 收取 usage 与 `[DONE]`，异常长尾会被主动终止；若旧模型仍开始输出 `deferred_ids`，识别到该字段前缀便立即停止，不读取数字列表。服务器即使忽略流式传输、直接返回普通 JSON，也可以解析其内容和顶层 `usage`；旧版完整 JSON 结构也保留兼容路径。

### 5. 校验、提交与回退

模型结果只有满足以下条件才会被采用：

- 每个当前坐标恰好覆盖一次，顺序不变，没有遗漏、重复或虚构 ID；
- 未完成内容只能是连续的末尾后缀；
- 语义单元不跨越硬时间边界，也不超过时长和文本大小安全上限；
- 对齐块在各语义单元内连续、完整，且译文非空。

扩展不再用数字出现次数、译文长度、文字体系、原文相似度、URL 或邮箱等启发式规则判断翻译质量；这些内容只由模型提示词约束，不会触发额外修复请求或阻断结构正确的译文。

即使模型认为窗口尾部已经完整，扩展也不会立即提交触碰末尾 16 坐标安全区的语义单元，而会把整个单元带入下一次更长窗口。每个硬边界区域只允许从左到右提交连续前缀，不能留下空洞，也不能让较晚返回的请求覆盖已经确认的结果。这正是跨 YouTube cue 的短语不会被固定批次边缘切开的关键。

如果任何 JSONL 行在首个有效单元之前就失败，扩展仍可尝试旧版完整对齐 JSON，再尝试较简单的整段语义 JSON。若错误、断网、超时或取消发生在若干有效行之后，已经逐行提交的连续前缀会保留，只有未提交后缀会重新请求；这样既不会重复覆盖旧结果，也不会因为坏尾部浪费前面已完成的译文。网络请求最多进行 3 次 HTTP 尝试，并处理超时、`429` 和临时服务器错误；一旦本次流已经提交有效单元，同一 HTTP 请求不会自动从头重放，以免产生重复行。

### 6. 播放、跳转、预加载与并发

连续播放时，当前范围优先，其后的范围按“预加载批数”提前处理。每个标签页通常最多同时保留 3 个 AI 请求，目的是避免多个推测性预加载把接口、费用和浏览器连接占满；正在观看或刚跳转到的位置属于紧急请求，即使预加载已经占满限额也可以优先通过。

拖动进度条到未缓存位置时，旧位置译文会立即清除并显示加载状态。距离很远的跳转会取消过时的焦点请求，从目标附近重新建立安全窗口：左侧坐标作为只读式边界判断材料，目标右侧同时预留独立的语义展开空间，最末端安全区不再与目标相邻。这样目标所在的完整语义单元通常能在首轮流式返回时直接提交；极长单元仍会继续扩窗。左边缘未确认的语义单元不会贸然显示，已经通过校验的缓存可以直接复用。

拿不到完整字幕轨而进入页面字幕回退模式时，同一时间只保留最新的一次翻译请求；字幕变化会真正取消旧请求，而不只是忽略旧响应。已验证的完整请求结果还会进入最多 96 项、约 2MB 的会话 LRU 缓存，缓存键包含接口、模型、思考模式、语言、上下文配置和完整源内容；刷新或再次遇到相同请求时可直接复用，浏览器会话结束后自动清除。

## 分页逻辑

翻译“语义单元”和屏幕“显示页”是两层概念：AI 决定哪些连续词属于同一句或分句，并给出源文—译文对齐块；浏览器再根据当前播放器的真实尺寸决定一次显示多少个对齐块。改变字体不会要求 AI 重新翻译同一句。

分页步骤如下：

1. 取字幕覆盖层可用宽度；通常接近播放器宽度的 98%，扣除内边距，并保证最小测量宽度为 260px。
2. 用 Canvas 按当前原文/译文字体、窗口或全屏字号分别测量实际像素宽度，而不是用字符数估算。
3. 每种语言以约 `可用宽度 × 1.68` 作为一页容量，给比例字体换行和内边距留出余量，目标是不超过约两行。原文和译文所需页数取较大者，且页数不会超过该语义单元拥有的计时坐标数。
4. 优先按 AI 返回的完整对齐块从左到右装页；加入下一个块会使任一语言超宽时才换页，因此专名、固定表达或完整分句只要处于同一对齐块就不会被分页拆开。
5. 若旧接口没有对齐块，或单个对齐块本身已经超宽，则进入像素感知后备分页：优先句末，其次分句标点、空格和中日韩字符边界；避免把冠词、介词、连词单独留在页尾。
6. URL、邮箱、连续缩写或首字母串、日期、时间、版本号、小数和省略号会先标记为不可从内部切开的形状。该保护基于通用文本形态，不维护具体名称或短语词典。
7. 原文页和译文页按语义顺序配对，再把每个计时坐标单调映射到页码。播放时间前进时只会向后翻页，不会在相邻 cue 间来回跳；同一语义单元中的短空隙最多可桥接 2200ms。
8. YouTube 的重叠 cue 可能让 cue 尾部的独立语义单元只剩几帧可见。若一个语义单元的真实可见时长不足 650ms，且它与相邻单元来自同一个原始 cue，显示层会将二者共同分页和显示。AI 的语义边界、译文缓存和请求内容不变，也不会因此增加 Token；不同原始 cue 绝不由这项规则合并。

以下变化会使用已有译文重新计算全部显示页：字体、窗口字号、全屏字号、显示/隐藏某一行、覆盖层宽度变化、浏览器窗口变化、进入或退出全屏，以及网页字体加载完成。颜色、背景、描边、上下顺序、行间距和垂直拖动位置不改变横向文字容量，因此不会触发分页，也不会发送新的 AI 请求。

## 隐私

无账号、无统计、无追踪。字幕文本只发送到你配置的 AI 接口；普通设置保存在 `chrome.storage.sync`，包含 API Key 与额外请求参数的连接配置保存在 `chrome.storage.local`、不会同步，已验证译文的有界缓存与接口报告的 Token 汇总保存在 `chrome.storage.session` 并随浏览器会话清除。
